作ったモジュール置き場（仮）。

プロジェクト単位でも使えると思う。

はう・とぅ〜・ゆ〜ず

  • ・DOMはスタイルガイドからコピーしてくればおｋ
  • ・Sassは/scss/styleguide/にある該当Scssファイル（見出しと同じ名前のファイル）からコピーしてくる。
  • 　※他の方法がないか考え中。スタイルガイドだけで完結できると良いよね。

ふぉ〜まっと

ルール

  • ・左端に余白は空けない
  • ・説明は総じて必ず記入する。（どんな条件でこの状態になるか...など）
  • ・亜流があるときはclass属性に {$modifiers} を書き込む
    <a href="#" class="button {$modifiers}">Link</a>
    ※これをやるとなぜか亜流が反映されるのです。

記入例


/*
#モジュールの説明
A button suitable for giving stars to someone.

#亜流の説明(ここに亜流を書くと{$modifiers}に反映される。)
:hover             - Subtle hover highlight.（ホバー時の説明）
.stars-given       - A highlight indicating you've already given a star.（stars-given付与時の説明）
.stars-given:hover - Subtle hover highlight on top of stars-given styling.
.disabled          - Dims the button to indicate it cannot be used.

#実際のDOM構造
Markup:
<a href="#" class="button {$modifiers}">Link</a>

#見出し
Styleguide 2.1.3.
*/
#モジュールのSass,CSSを記入する。
a.button.star{
  ...
}
a.button.star.stars-given{
  ...
}
a.button.star.disabled{
  ...
}

これで一つのモジュールのスタイルガイドが完成。

モジュールを新規作成するときは...

既存のパーツ群に追加する場合は...

ex）新しいグロナビ作ったで！ -> /scss/styleguide/ に _nav.scss がある！ -> じゃあそこに追加や！
  • ・既にあるモジュールの下に一行空けて自分のモジュールを追加する。

既存のパーツ群がない場合は...

ex）新しいグロナビ作ったで！ -> /scss/styleguide/ に _nav.scss がない！ -> じゃあ新しく _nav.scss 作るで！
  • 1. /scss/styleguide/ に _nav.scss を作成。
  • 2. /scss/styleguide/ の styleguide.scss に _nav.scss を追加。
  • 3. /styleguide/ の index.html のリストに新規追加したパーツ群を追加。
